This is a progress tracker for a Lean 4 formalization of stabilizer-formalism quantum error-correcting codes (QEC). The codes themselves come from the Error Correction Zoo — a community-maintained catalog of ~267 quantum codes. Formalization here means proving every claim about a code — parameters [[n,k,d]], stabilizer generators, logical operators, code distance — inside a proof assistant, so the end artifact is machine-checked mathematics rather than an informal write-up.

The pipeline scores every catalog entry on six axes (reuse: how directly existing infrastructure applies; canonicality: textbook/syllabus weight; hardware: appearance in published hardware demos; tractability: whether a clean distance proof is known; prerequisites: how much missing infrastructure stands in the way; effort: cheapness to formalize). Each axis is graded 0–10 and combined into a single composite score:

composite = 0.25·reuse + 0.15·canon + 0.15·hardware + 0.15·tract + 0.20·prereq + 0.10·effort

The composite sorts each code into one of four tracks: engineering (known proofs — let the agents grind) · moonshot (novel research — distance proof is the open problem) · defer (blocked on missing infrastructure) · skip (not worth pursuing in this repo). Claude Code agents draft Lean skeletons and close sorry markers; a human reviews at two gates — spec review (catch wrong theorem statements before proof effort) and PR review (final code).

Reading this dashboard:  Overview — counts, in-flight, queue head  ·  Queue — all 267 codes, sortable  ·  Research — moonshots; failures are first-class outputs
