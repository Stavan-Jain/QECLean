# Catalog

Structured metadata about quantum error correcting codes, ingested from the
[Error Correction Zoo](https://errorcorrectionzoo.org/) and scored for
formalization priority.

## Files

- `zoo.yaml` — ingested catalog (~200 quantum codes). One entry per code with
  `{code_id, short_name, family, params, original_refs, related_codes,
  description, source_path}`. Generated by `scripts/ingest_zoo.py`.
- `scoring.yaml` — per-code formalization-priority scores along six axes:
  reuse, canonicality, hardware, tractability, prerequisites, effort.
  Generated by the `qec-prioritizer` agent.

Both files are checked into the repo so the queue is reproducible.

## Refresh cadence

- `zoo.yaml`: regenerate when the eczoo_data SHA pin is bumped (see
  `pipeline/cache/PIN.md`).
- `scoring.yaml`: regenerate on every major repo milestone (new abstraction,
  new code completed) — reuse scores change as the repo grows.
