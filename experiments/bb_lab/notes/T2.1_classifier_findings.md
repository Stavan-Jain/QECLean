# T2.1 — Tightness classifier findings

**Goal.** The textbook CSS upper bound
`d_X ≤ min(d_A^⊥, d_B^⊥) ≡ min(min_wt_ker_A, min_wt_ker_B)` is tight
22.4% globally (859/3836) but per-group rates range 10.7% (`Z5xZ6`) to
90.6% (`Z4xZ6`). Hunt for a structural condition `C(G, A, B)` that
predicts tightness.

## Corpus and target

3 836 rows from `bb_instances.duckdb` with both `d_exact` and
`min_wt_ker_{A,B}` populated. Target

    tight ≡ (d_exact == min(min_wt_ker_A, min_wt_ker_B))

Class balance: 859 tight / 2 977 loose ≈ **22.4 %** baseline.

Per-group tightness (sanity check, matches HANDOFF.md):

| group   |     n | tight |  rate |
|---------|------:|------:|------:|
| Z3xZ3   |    12 |     6 | 50.0% |
| Z3xZ4   |    73 |    28 | 38.4% |
| Z3xZ5   |   103 |    13 | 12.6% |
| Z3xZ6   |   166 |    80 | 48.2% |
| Z4xZ6   |   106 |    96 | **90.6%** |
| Z5xZ6   | 2 622 |   280 | **10.7%** |
| Z6xZ6   |   754 |   356 | 47.2% |

## Classifier (decision tree, depth 4)

Features: `n, k, A_weight, B_weight, rank_HX, rank_HZ, dim_ker_A,
dim_ker_B, orbit_size, tanner_girth, supp_diameter_{A,B},
min_wt_ker_{A,B}, ell, m`, plus derived
`min_minwt, max_minwt, minwt_gap, minwt_eq`, plus one-hot
`group_struct`. (`A_weight = B_weight = 3` across the entire corpus, so
those columns are constant and dropped.)

Trained with `class_weight="balanced"`, `random_state=42`. Train
accuracy 98.7 %; **5-fold CV accuracy 98.6 % ± 0.5 %**.

Confusion matrix (rows = true [loose, tight], cols = predicted):

    [[TN = 2927   FP =   50]
     [FN =    0   TP =  859]]

Recall on the tight class is 100 % (the tree never misses a tight code);
50 loose codes are mis-classified as tight (false positives). These 50
are the `min_minwt ≤ 4 AND n/k < 4` exceptions discussed in §Candidate
conditions below.

    |   |   |--- dim_ker_B <= 10.00
    |   |   |   |--- tanner_girth <= 5.00
    |   |   |   |   |--- class: 0 (loose)
    |   |   |   |--- tanner_girth >  5.00
    |   |   |   |   |--- class: 1 (tight)
    |   |   |--- dim_ker_B >  10.00
    |   |   |   |--- orbit_size <= 3456.00
    |   |   |   |   |--- class: 0 (loose)

So the tree is essentially a threshold rule **`min_minwt ≤ 5 → tight`**,
with all the depth-3/4 splits doing minor cleanup around the

| feature         |   coef |
|-----------------|-------:|
| `dim_ker_A`     |  +3.90 |
| `dim_ker_B`     |  +2.76 |

the same direction (small `min_wt_ker_A`, small `min_minwt` → tight).

## Candidate conditions

Each row of the following table is an explicit boolean predicate on
`(G, A, B)`-derived features. Rate = `#tight / #applicable`.

| C    | predicate                                       | applicable | tight | rate |
|------|-------------------------------------------------|-----------:|------:|------:|
| C_A  | `min_minwt ≤ 4 AND n ≥ 4k`                      |    **692** |   692 | **100.00 %** |
| C_B  | `min_minwt ≤ 4`                                 |        774 |   735 |  95.00 % |
| C_C  | `min_minwt ≤ 6`                                 |      1 008 |   859 |  85.22 % |
| C_D  | `min_minwt ≤ 6 AND n ≥ 4k`                      |        920 |   816 |  88.70 % |
| C_E  | `min_minwt ≤ 4 AND n ≥ 2k`                      |        759 |   720 |  94.86 % |

Sanity check: `min_minwt ≥ 8` ⇒ 0 / 2 828 tight (the textbook bound is
never tight on this regime in our corpus).

**Headline.** `C_A := min_minwt ≤ 4 AND n ≥ 4k` is **100 % tight on
692 / 3 836 = 18 % of the corpus**, and across every group containing
applicable rows (`Z3xZ3, Z3xZ4, Z3xZ5, Z3xZ6, Z4xZ6, Z5xZ6, Z6xZ6`).
This is the strongest candidate.

Failure mode of `C_B` (the simpler version): exactly the 39 codes with
`min_minwt = 4`, `d_exact = 2`, and `n / k = 3` (high-rate weight-3 BB
codes at `Z3xZ3 → Z6xZ6` with `dim_ker_A = dim_ker_B = k / 2`). These
codes have a logical of weight 2 that the textbook CSS bound misses by
2. The `n ≥ 4k` cut excludes exactly that band.

## Z4xZ6 vs Z5xZ6 explanation

`Z4xZ6` and `Z5xZ6` are not "structurally different in a way the
classifier picks up" — the classifier picks up `min_minwt` regardless
of group. The difference is **purely distributional**:

* **`Z4xZ6`** (`n = 48, k = 16`): every instance has `min_minwt ∈ {2, 4}`.
  At those values the CSS bound is essentially always tight (101 / 106
  = 95 % at `min_minwt = 4`; 5 / 5 at `min_minwt = 2`). So the group is
  ~100 % tight not because of `Z4xZ6`-specific structure but because
  weight-3 polynomials over `Z4 × Z6` happen to all yield small dual
  weights.
* **`Z5xZ6`** (`n = 30 / 60 / 90 / 120`): `min_minwt` ranges over `{2,
  4, 8, 10, 16, 20}`. The 82 + 206 = 288 instances at `min_minwt ≤ 4`
  are 97 % tight — same regime as `Z4xZ6`. The other 2 334 instances
  (`min_minwt ≥ 8`) are 0 % tight, dragging the group rate to 10.7 %.

So the **structural** content of the apparent Z4 / Z5 split is: weight-3
polynomials over `Z_a × Z_b` with `min(a, b) ≤ 4` tend to have `ker(M_A^T)`
and `ker(M_B^T)` saturate to small minimum weight (cyclic-code reason —
short cyclic blocks have short dual minimum weights), whereas larger
`min(a, b)` lets the dual minimum weight grow above the actual code
distance.

This is **not** a `Z4`-vs-`Z5` algebraic phenomenon. The classifier
correctly extracted `min_minwt` as the universal driver; the group is a
proxy for "what `min_minwt` values are reachable".

## Caveats — what the data does NOT say

1. **`A_weight = B_weight = 3` everywhere in the corpus.** All
   conjectures below are about the weight-3 BB regime only. Behavior
   at weight 4 / 5 / 6+ (where the IBM gross polynomial lives) is
   unmeasured.
2. **`d_exact ∈ {2, 4, 6, 8}` only.** All instances are at most n = 108
   (most are n ≤ 72). Whether `min_minwt ≤ 4 AND n ≥ 4k` continues to
   be 100% at `d_exact = 10, 12, …` (the regime that includes gross
   `[[144, 12, 12]]`) is **out of distribution**. None of these
   candidates have been tested against a `d ≥ 10` code.
3. **The classifier sees `d_exact` indirectly via `min_minwt`** — the
   only reason a candidate condition can predict "tight" is that
   `min_minwt = d_exact` defines tightness and small `min_minwt` is a
   forced sandwich (`d_exact ≥ 2` in this corpus, CSS ≤ 2 when
   `min_minwt = 2`). C_A's content is non-trivial only for
   `min_minwt = 4`; everything at `min_minwt = 2` is a forced tie.
4. **Novelty is not claimed here.** The textbook bound itself is
   classical CSS dual theory (see §6a of HANDOFF.md). The candidate
   conditions narrow down *when* the bound is sharp; the literature
   side (T2.3) needs to confirm whether `min_minwt ≤ 4 AND n ≥ 4k → d
   = min_minwt` is in Lin–Pryadko 2306.16400, Kovalev–Pryadko 2013, or
   the BB-code surveys.
5. **5 0 false positives from the depth-4 tree.** All in the
   `min_minwt = 4, n = 3k` band excluded by C_A. The CV accuracy
   (98.6 %) is dominated by the easy `min_minwt ≥ 8 → loose` half.
