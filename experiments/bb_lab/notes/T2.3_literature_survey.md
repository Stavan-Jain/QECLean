## Notation map

The papers use overlapping but not identical notation. We use the
following bb_lab conventions to translate:

| bb_lab / corpus column   | paper notation (varies)                          |
|--------------------------|--------------------------------------------------|
| `n` (qubits)             | `N = 2ℓm` (Bravyi) / `N = 2n` (KP13)             |
| `k` (logical qubits)     | `K = 2 dim(ker A ∩ ker B)` (Bravyi Lemma 1)      |
| `d_exact`                | `d`, `d_Z`, `D` (KP13)                           |
| `A`, `B`                 | `a, b ∈ F[G]` (LP23) / `A = L(a)`, `B = R(b)`    |
| `min_wt_ker_A`           | `d(C_A^⊥) = d_A^⊥` (LP23) — the dual classical   |
| `min_wt_ker_B`           | `d(C_B^⊥) = d_B^⊥` (LP23)                        |
| `dim_ker_A`              | `k_1` (KP13) / dual code dim                     |
| `ell, m`                 | `|G|`, `ℓ` (lift size), circulant size           |

Note: `min_wt_ker_A` IS already the dual-classical distance `d_A^⊥`
because `ker(A) = ker(L(a))` (when read as the codespace of the
parity-check `A^T`) is the dual code `C_A^⊥`. The existing
`CSS_classical_dual` candidate in `tier2_explore.py` is named
correctly.

## 1. Lin–Pryadko 2023 — arXiv:2306.16400 "Quantum two-block group algebra codes"

The canonical 2BGA reference. Subsumes BB codes as the abelian-2BGA
case. Quotes here are from the v1 PDF; section refs match.

### Statement 3 (§III.C) — chain of upper bounds via shortening

> `d_Z ≡ d_Z(Q) ≤ d_Z(Q'_μ) ≤ d_Z(Q''_μ) ≤ d(C_μ)`, for `μ ∈ {L, R}`.
>
> [Eq. 30]

This is **a Z-shortening lemma chain**: the original two-block
code's distance is upper-bounded by the distance of a classical
code with parity-check matrix `H_μ` (rows of `A` or `B`,
respectively).

**Form in our notation**: `d_X ≤ d(C_A) = d_A` and similarly
`d_X ≤ d_B`, *but* `d(C_A)` here is the *code with parity-check
A* — i.e., `ker(A)` as a code. This is precisely `min_wt_ker_A`.
So **Statement 3 reduces to the textbook bound**
`d_X ≤ min(d_A^⊥, d_B^⊥)` already encoded as `CSS_classical_dual`
in `tier2_explore.py`. Lin-Pryadko explicitly says this in §III.C:

> "the usual CSS bounds, `d_Z ≥ d(C_{H_X}^⊥)`, `d_X ≥ d(C_{H_Z}^⊥)` [Eq. 31].
> However, since the rows of H_X and H_Z are mutually orthogonal, we have,
> e.g., `d(C_{H_X}^⊥) ≤ d(C_{H_Z}) ≤ W_Z`, the minimum row weight of the
> matrix H_Z. Since our main interest is in highly-degenerate quantum
> LDPC codes ... the CSS bounds (31) are not very useful."

So **Lin–Pryadko explicitly bounds `d_X` from above by the
classical dual distances and notes the lower-bound form is loose**.

### Statement 4 (§III.C) — block-diagonal A gives `d ≤ m`

> "Suppose matrix A is block-diagonal with the maximum block size m,
> and the code Q''_L is non-trivial, k(Q''_L) > 0. Then the distance
> `d_Z(Q''_L) ≤ m`."

This is specific to the block-diagonal case (Lin-Pryadko Statement
8 / the abelian-2BGA case): it gives `d ≤ min(|G_a|, |G_b|)` where
`G_a, G_b` are the support groups (cf. §III.C, just after Stmt 10:
"d ≤ min(|G_a|, |G_b|)"). Computable from corpus features
once support groups are extracted.

### Statement 5 (§III.C) — single-block-erasure lower bound

> "Suppose both rank defects in Eq. (22) are zero, δ_X = δ_Z = 0.
> Then, `d ≥ d_S`, `d_S ≡ d(A, B^T)` [Eq. 33]."

`d(A, B^T)` is the Z-distance of the **single-block erasure
subsystem code** `CSS(A, B^T)`. For BB (abelian) codes, the
condition δ_X = δ_Z = 0 always holds (Statement 9). So this gives a
non-trivial lower bound — but `d(A, B^T)` is itself a per-instance
quantum distance computation, NOT a closed-form feature of the
polynomials. **Not directly computable from the current corpus**;
it would require running another SAT distance solve at half the
qubits. Plausibly worth adding to the corpus as a future
`single_block_erasure_d` feature, but cost-wise basically equivalent
to computing `d_exact`.

### Statement 12 (§IV.F) — Tillich–Zémor analog (quasi-abelian LP)

> "Given elements `a, b ∈ F[G]` such that the intersection subgroup
> `N ≡ G_a ∩ G_b` of rank `c` is abelian and normal in both support
> groups, let `d_A^⊥` and `d_B^⊥` be the distances of classical
> F-linear group algebra codes with parity check matrices A = L(a)
> and B = R(b). Then the distance d_Z of the code LP[a, b] satisfies
> `d_Z ≥ d_0 ≡ ⌈min(d_A^⊥, d_B^⊥) / c⌉`."

This is the canonical Tillich–Zémor-style **lower bound**. The
"Theorem 5 from Ref. 14" the title refers to is the Kovalev–Pryadko
2013 hyperbicycle Theorem 5 (see §3 below). Key bound:

- **Lower bound**: `d ≥ ⌈min(d_A^⊥, d_B^⊥) / c⌉`
- where `c = |G_a ∩ G_b|` (rank of intersection subgroup).
- Conditions: N is abelian and normal in both G_a, G_b. **Always
  satisfied for abelian G** (which is the BB case).

When `c = 1`, the bound saturates to `d_Z = min(d_A^⊥, d_B^⊥)`
exactly — but `c = 1` is rare in engineered BB codes
(gross: c=24 → d_0 ≤ 3, see `pipeline/attempts/gross/result.md`).

**Form in our notation**: `c = |⟨supp(A)⟩ ∩ ⟨supp(B)⟩|`. NOT in
corpus today; could be added as a feature
(`intersection_subgroup_order`). Computable from `(G, A, B)`.

### Statement 13 (§IV.F) — Tillich–Zémor analog upper bound

> "Let J be a maximal ideal in F[N], C_J the two-sided coset code
> generated by J, and Ĉ_J ≡ P C_J its image under the linear map (5).
> Denote d' the distance of the subcode `C_A^⊥ ∩ Ĉ_J`. Then, if
> `C_{B^T}^⊥ ∩ Ĉ_J ≠ {0}`, the distance of the 2BGA code LP[a, b]
> satisfies `d_Z ≤ d'`."

This is a **conditional upper bound** through a maximal-ideal coset
code intersection. Heavy machinery; not corpus-computable without
implementing maximal-ideal enumeration in `F_2[N]`.

### Lin–Pryadko corollary at c=1 (§IV.F, eq. after Stmt 13)

> "The upper and the lower bounds on d_Z coincide when c = 1: in
> this case the subgroup N = {1} is trivial so that F[N] is just
> the field F, and the auxiliary codes in statements 12 and 13
> coincide, which gives `d_Z = min(d_A^⊥, d_B^⊥)`."

**Tightness condition**: c = 1 ⟹ bound is tight. The gross code
has c = 24, so this is hopeless for gross. For our corpus we
should check whether the c=1 instances have d = min(d_A^⊥, d_B^⊥)
exactly.

## 2. Bravyi–Cross–Gambetta–Maslov–Rall–Yoder 2024 — arXiv:2308.07915 "High-threshold and low-overhead FT QM"

The IBM gross-code paper. The BB construction is presented in §IV
and the distance methodology in §V.

### Distance derivation: MIP, not analytic

§V Table 3 caption, line 564:

> "Code distance was computed by the mixed integer programming
> approach of Ref. [55]. Notation ≤ d indicates that only an upper
> bound on the code distance is known at the time of this writing."

**Bravyi et al. derive NO analytic distance bound**. Every entry in
their Table 3 is computer-search verified. The `[[144, 12, 12]]`
gross code's distance is established by MIP, not formula.

### Lemma 1 — basic parameters

> `n = 2ℓm`, `k = 2 · dim(ker(A) ∩ ker(B))`, `d = min{|v|: v ∈ ker(H_X) \ rs(H_Z)}`
>
> The code offers equal distance for X-type and Z-type errors.

The d_X = d_Z equality is proved via a permutation conjugation
(matrix C with C A C = A^T). Not a *bound* but a structural lemma.

### Citations of prior bound work

Bravyi et al. cite:
- [35] = Kovalev–Pryadko 2013 (BB codes are quasi-cyclic GB codes — origin)
- [54] = Panteleev–Kalachev 2021 (BB codes are a special case of LP codes)
- [55] = MIP-based distance algorithm

They do NOT cite Lin–Pryadko 2306.16400 (the Lin–Pryadko paper
postdates the Bravyi v1 by ~2 months). They do not derive or
explicitly state `d ≤ min(d_A^⊥, d_B^⊥)` — they use MIP instead.

## 3. Kovalev–Pryadko 2013 — arXiv:1212.6703 "Quantum hyperbicycle LDPC codes"

The original quasi-cyclic / hyperbicycle construction. Phys. Rev. A
88, 012311. Distance bounds in §IV.

### Theorem 2 — generalized bicycle distance lower bound

> "The generalized bicycle codes in Eq. (18) have the block length
> N = 2n, the number of encoded qubits K = 2 deg p(x) + 2 deg r(x) − 2n
> ... and the distance exceeding or equal that of the classical code
> over F4 formed by codewords orthogonal to G with respect to the
> trace inner product."

This is a lower bound through a classical-cyclic-code embedding.
Computable in principle (compute the classical F_4 code's minimum
distance), but BB codes are bivariate so this 1D bound doesn't
directly apply. Note Stmt 2 is **only tight for pure (non-degenerate)
codes**.

### Theorem 3 — hyperbicycle lower bound (the Tillich–Zémor analog)

> "A quantum CSS code with generators (19) [hyperbicycle] satisfies
> `D ≥ min(d_1, d_2, d̃_1, d̃_2)`."

This is the canonical TZ lower bound. Note the c=1 special case here
matches the hypergraph-product bound (KP §IV.A discussion).

For BB (special case χ=1 in the hyperbicycle ansatz), `c` is the
order of the intersection subgroup as in LP23 Stmt 12; the χ=1
specialization gives the abelian-LP case.

### Theorem 5 — general hyperbicycle distance lower bound

> "The minimum distance of the code with generators (19) satisfies
> the lower bound `D ≥ ⌊d / c⌋`, `d ≡ min(d_1, d_2, d̃_1, d̃_2)` [Eq. 32]."

This is `D ≥ ⌊min(d_A^⊥, d_B^⊥) / c⌋` in the BB notation, i.e.
**identical to Lin–Pryadko Statement 12** modulo a floor-vs-ceiling
quibble (LP uses ⌈⌉, KP uses ⌊⌋; the difference is a tiny
discretization, not a substantive bound difference).

### Theorem 6 — hyperbicycle distance upper bound

> "For every p(x), a binary factor of x^c − 1 such that k_1^(p) > 0
> and k̃_2^(p) > 0, the minimum distance D of the code with generators
> (19) satisfies `D ≤ min(d_1^(p), d̃_2^(p))`. Similarly, when k_2^(p) > 0
> and k̃_1^(p) > 0, we have `D ≤ min(d_2^(p), d̃_1^(p))`."

This is the Tillich–Zémor upper bound through factor polynomials of
x^c − 1; for BB the analog is via maximal ideals of `F_2[Z_ℓ × Z_m]`.
For computational use, this requires polynomial factoring.

### Concise restatement combining Thm 5 & 6 (from §IV.D end):

> "D limited by `⌊d/c⌋ ≤ D ≤ d`, `d ≡ min(d_1, d_2, d̃_1, d̃_2)` [Eq. 38]."

The classical-dual upper bound is `d` itself; the lower bound is
`⌊d/c⌋`. The interval shrinks to a point when c = 1, matching
LP23's tightness condition.

## 4. Panteleev–Kalachev 2021 — arXiv:2012.04068 "Quantum LDPC with almost linear min distance"

Lifted-product codes; BB is the special case where the ring is
`R = F_2 G` for G abelian.

### Theorem 1 — existence of GHP codes with Θ(N/log N) distance

> "There exists a family of QLDPC codes of dimension Θ(log N) and
> distance Θ(N / log N) as the code length N → ∞."

Constructive but uses **expander-graph base matrices A** (Theorem
1.2 from Ref. 22). Not deterministic for a given (G, A, B); not
applicable to our gross-style BB codes which are NOT random expander
constructions.

### Theorem 2 — existence of LP codes parametrized by α

> "For every α such that 0 ⩽ α < 1 there exists a family of QLDPC
> codes of dimension Ω(N^α log N) and distance Ω(N^(1−α/2) / log N)."

Same caveat: existence theorem, not computable bound.

### Quasi-abelian decomposition (§III.E + Appendix B)

For abelian G and odd `|G|` (Maschke applies), the QA LP code
decomposes into HP codes per character:

> `dim LP(A, B) = Σ r_i dim HP(A_i, B_i)`, where `q_i = 2^{r_i}` and
> the sum runs over the irreducible factors of F_2 G as a product
> of finite fields.

**This is the algebraic side of the character decomposition** and
gives `k` (dimension), not `d`. **Important note**: for BB codes
where 2 divides |G| (which includes gross with |G| = 72 = 8·9),
the algebra `F_2 G` is **not semisimple**, so this decomposition
fails directly and one needs the Jacobson-radical refinement.

### No computable-from-(G,A,B) deterministic distance lower bound

Panteleev-Kalachev does NOT provide a deterministic bound of the form
`d ≥ f(G, A, B)` for the BB / quasi-abelian case. Their proof is
asymptotic-existence with expander A.

## 6. Raveendran–Declercq–Vasić 2025 — arXiv:2503.07567 "On the Minimum Distances of Finite-Length LP QLDPC Codes"

Recent finite-length LP-QLDPC distance analysis. Highly relevant.

### Theorem 1 — orthogonality conditions for LP-QLDPC

Conditions on the quasi-cyclic base matrices `B_X`, `B_Z` such that
`H_X H_Z^T = 0`. Structural, not a distance bound.

### Theorem 2 — RCPC-based distance characterization

> "Given a quasi-cyclic base matrix B with minimum distance d_C^min
> and of size m × n of an LP-QLDPC code, if there exists partitioning
> of rows or columns such that the row/column partitioning constraint
> (RCPC) is satisfied, then the LP-QLDPC code has a minimum distance
> `d_Q^min = min(d_C^min, m + n)`."

**Form in our notation**: For BB codes (m = n = ℓm for a single
"super-block"), this would give `d ≤ min(d_C^min, 2ℓm)` *under*
the RCPC condition.

But: BB codes use a specific 2×1 LP structure (left-Right blocks
each circulant), not the general protograph LP-QLDPC of Raveendran.
The **RCPC condition** is specific to type-I protographs and may
or may not apply to BB. Worth checking on the corpus —
specifically, whether the BB structure satisfies an analog of the
row-partition condition `r_1 + r_m = r_2 + r_(m-1) = ...`.

Corollary 1: For base matrix with m = 2 (or n = 2), RCPC is
automatically satisfied, giving `d_Q ≤ n + 2`.

**Caveat**: this is a `d ≤` upper bound, not a lower bound. And
it requires a non-trivial structural check (RCPC) to fire.

## 8. Wang–Pryadko 2022 — arXiv:2203.17216 "Distance bounds for GB codes"

GB codes are the 1D-cyclic special case of BB (`m = 1` essentially).

### Statement 2 — `d ≤ d_g` via cyclic-code embedding

> "The Z-distance of the AB code CSS(H'_X, H_Z) satisfies `d'_Z ≤ d_g`,"
> where `d_g` is the distance of the cyclic code with generating
> polynomial `g(x) = (x^ℓ − 1) / gcd(a(x), b(x), x^ℓ − 1)`.

For BB this would generalize to a multivariate gcd of (A, B, x^ℓ - 1, y^m - 1)
but the standard cyclic gcd machinery is 1D. **Not directly applicable** to
bivariate BB without nontrivial extension.

### Statement 13 — D-dimensional locality upper bound

> "An incommensurate GB code with row weight w and parameters [[n=2ℓ, k, d]]_q
> is equivalent to a CSS code local in D ≤ w − 1 dimensions" with bounds
> `d ≤ O(n^(1−1/D))` and `k · d^(2/(D−1)) ≤ O(n)`.

For BB with row weight w = 6 (gross), D ≤ 5 ⟹ `d ≤ O(n^(4/5))`. For
n = 144 this gives `d ≤ O(50ish)` (asymptotic, big-O constants matter).
**Form**: corpus-computable as an asymptotic estimate, but the
multiplicative constants are unspecified, so this isn't a sharp
numerical bound. Could be encoded as a **weak-asymptotic candidate**
for the corpus (`d ≤ n^(1 − 1/(w − 1))`).

## 10. Calderbank–Shor 1996 / Steane 1996 — CSS distance basics (textbook)

For ANY CSS code (not BB-specific):
- `d_X = min |c|` over `c ∈ ker(H_Z) \ rowspan(H_X)`
- `d_X ≥ d(C_X^⊥)` where `C_X^⊥` is the dual classical code of `H_X`
- Equivalently `d_X ≤ min row-weight of H_Z` (since rows of H_Z lie
  in the dual code).

For BB:
- `H_X = [A | B]`, `H_Z = [B^T | A^T]`.
- Any nonzero `f ∈ ker(A^T) = ker(M_B^T) = C_A^⊥` gives X-error
  `(f, 0)`, which satisfies `H_Z (f, 0)^T = 0`. So `(f, 0)` is at
  worst a stabilizer; if it's not in rowspan(H_X), it's a nontrivial
  X-logical of weight `|f|`. Hence `d_X ≤ |f|` for any such `f`,
  i.e. `d_X ≤ d_A^⊥`. Symmetric for B.

**Form in our notation**: `d ≤ min(min_wt_ker_A, min_wt_ker_B)` =
`min(d_A^⊥, d_B^⊥)`. **THIS IS THE TEXTBOOK CSS BOUND. NOT NOVEL.**

## Computable from our corpus today (Tier-2 candidates)

The following bounds are literature-cited, BB-applicable, and
expressible in (or close to) corpus columns currently populated.

1. **CSS dual upper bound**: `d ≤ min(min_wt_ker_A, min_wt_ker_B)`.
   Already in `tier2_explore.py` as `CSS_classical_dual`. Citation:
   CSS-theory + Lin-Pryadko §III.C Stmt 3.

2. **Check-weight upper bound**: `d ≤ A_weight + B_weight`.
   Already in `tier2_explore.py` as `check_weight_sum`. Trivial.
   Citation: textbook (any CSS row of `H_X` IS a stabilizer ⟹ row
   weight ≥ d_X for any non-trivial logical).

3. **TZ lower bound (Stmt 12 / Thm 5)**: `d ≥ ⌈min(d_A^⊥, d_B^⊥) / c⌉`.
   NEW candidate, requires `c = |G_a ∩ G_b|` to be a column. Not
   currently in the corpus, but easy to add via the existing
   `Aut(G)` / support-group infrastructure (see `automorphism.py`).
   Tracked as **TZ_lower_bound** candidate below.

4. **Support-group ceiling**: `d ≤ min(|G_a|, |G_b|)`. NEW candidate,
   requires support-group orders. Computable from `A_poly`, `B_poly`
   via the existing `group.py` infrastructure. Tracked as
   **support_group_ceiling**.

5. **Wang-Pryadko locality asymptotic**: `d ≤ n^(1 − 1/(w−1))`.
   Weak asymptotic; numerical interpretation requires multiplicative
   constants that the literature does not pin down. Included with a
   loose-constant version (`d ≤ n^(1 − 1/(w−1))`).

6. **Single-block erasure lower bound**: `d ≥ d(A, B^T)`. NOT
   currently computable without running an additional half-size SAT
   distance solve. Skipped at this layer — needs corpus extension.

## Bounds NOT in the literature (to my knowledge after this survey)

After this survey, I am unaware of any literature bound of the form

> `d ≥ f(G, A, B)` for some closed-form `f` depending on
> polynomial features (gcds, Newton polytopes, character isotypes,
> etc.) that is **non-asymptotic, deterministic, and tight on
> engineered BB codes**.

The closest candidates that DO exist (Stmt 12 / Thm 5 / Camion
multivariate / Stmt 5) are *provably loose* on the gross polynomial
pair (see `pipeline/attempts/gross/result.md`: TZ ≤ 3, Camion ≤ 9
for d_true = 12).

Note: I am being cautious about novelty. "Bounds NOT in the
literature" here means "I did not find them in the five papers I
read plus the secondary literature in
`pipeline/attempts/gross/literature_notes.md`". The actual research
question for Tier 2 is whether **any new f(G, A, B)** can be
identified by the conjecture mill against the corpus that is both:
1. Tight on the gross polynomial pair (d_pred ≈ 12)
2. Cheap to compute relative to MIP

Possible angles (none of which I'm claiming novel; all need
further literature checking before promoting beyond hypothesis):
- Refinements via Fourier isotypes over `F_q[G]` for `q` such that
  `q ≡ 1 (mod |G|)` (the Bernal–Bueno-Carreño–Simón apparent
  distance machinery applied to F_4-2BGA codes — see
  `pipeline/attempts/gross/literature_notes.md` § Camion).
- Polynomial-GCD-based bounds in `F_2[Z_ℓ × Z_m]` (Otjens uses
  Gröbner bases for `k`; same machinery has not been applied to
  `d` per Otjens's own statement of openness).
- Cover-graph chain-map transfer bounds (Pesah et al. 2025): if
  the gross code is an h-cover of a base BB code with known
  distance, `d_gross ≥ d_base` holds.

## Tightness conditions extracted from the literature

| Bound                          | Tight when                                                | Loose when                       |
|--------------------------------|-----------------------------------------------------------|----------------------------------|
| CSS dual `d ≤ min(d_A^⊥, d_B^⊥)` | "Pure" codes; many small BB cases (need corpus check)    | Degenerate codes (gross? check)  |
| TZ lower `d ≥ d_0/c`           | `c = 1` (Lin-Pryadko §IV.F explicit)                      | `c > 1` (typical for engineered BB) |
| KP13 Thm 2 (GB lower)          | Non-degenerate / "pure" codes                             | Highly degenerate (KP says so)   |
| Wang-Pryadko Stmt 13 locality  | (asymptotic; no finite-N tightness claim)                 | always loose by big-O constants  |
| PK Thm 2.18 (Arnault et al.)   | (asymptotic; same caveat as W-P)                          | always loose by big-O constants  |

**Strategic implication**: BB codes engineered for high distance
(gross-style) **specifically violate the tightness conditions** of
all known analytic bounds. This is consistent with the moonshot's
core observation: algebraic bounds are structurally loose on
engineered BB codes.

## Open questions the literature flags about BB-code distance

These are explicit "this is open" statements in the literature, by
paper. Potential Tier-2 / Tier-3 targets.

1. **Postema–Kokkelmans arXiv:2502.17052 abstract** (verified
   verbatim, A1 lane L3): "Recently, bivariate bicycle (BB) codes
   have emerged as a promising candidate for such compact memory,
   though the exact tradeoff of the code parameters [[n,k,d]]
   remained unknown." This is the headline open question.
   *(Correction 2026-06: this item previously quoted an "Otjens 2025
   abstract" saying "no closed-form analytical formula relates d to
   the polynomial exponents" — that quote appears in neither
   arXiv:2502.17052 nor the Otjens BSc thesis; it was apocryphal.)*

2. **Lin-Pryadko 2306.16400 §IV.F**: the c=1 case is solved (TZ
   bound is tight there). The c > 1 case has only the upper-lower
   gap `⌈d_0/c⌉ ≤ d ≤ c · min(ℓ_a, ℓ_b)`. **Closing this gap
   for c > 1** would be a publishable result.

3. **Kovalev-Pryadko 2013 §IV.D**: "The codes in the last two
   examples exceed the lower bound in Theorem 2 due to degeneracy."
   I.e. **degeneracy gap on Thm 2** is open in general — the bound
   only equals the true distance for "pure" (non-degenerate) codes.

4. **Bravyi 2024**: distances `[[360,12,≤24]]` and `[[756,16,≤34]]`
   are listed with `≤`, meaning the MIP-derived upper bounds are
   not yet matched by lower-bound certifications. **Tight analytic
   lower bound for these would be a contribution to the BB-code
   literature.**

5. **Panteleev-Kalachev 2021 §III.E**: the quasi-abelian
   semisimple-character decomposition is only available when
   `gcd(|G|, char F) = 1`. For BB codes over `F_2` and `|G|` even
   (which includes gross), the **modular case (Jacobson-radical
   refinement)** is open — Panteleev-Kalachev punts on it.

6. **Raveendran-Declercq-Vasić 2025**: their Theorem 2 (RCPC-based
   distance characterization) applies to type-I protograph LP-QLDPC
   codes. **Whether BB codes satisfy the RCPC** (and under what
   conditions) is not addressed in their paper — could be a corpus
   check.
