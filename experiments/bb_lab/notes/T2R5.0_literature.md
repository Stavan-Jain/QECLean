# T2R5.0 — Literature lookup: homological / cover-graph chain-map bounds

**Conclusion (preview).** The relevant homological-bound paper is
**Symons–Rajput–Browne 2025** ([arXiv:2511.13560](https://arxiv.org/abs/2511.13560)),
not the spec-listed "Pesah–Roffe 2025" (which is not a real paper title — it
appears to be a misremembering of the author of an earlier work that
*motivated* the cover-graph approach but did not formalize a BB distance
bound). SRB 2025 is the **direct, BB-code-specific** application of
cover-graph chain-map transfer to BB code distance, published Nov 17, 2025.

SRB 2025 **does not require F₂[G] semisimple**; its derivation is on
chain complexes, so the §6j obstruction does *not* directly apply. **But
the gross code [[144,12,12]] is a double cover (h=2) of [[72,12,6]], and
all SRB rigorous bounds on `d` require `h` odd.** The proof technique
collapses for even `h` because `Lemma 4.4` gives `p ∘ τ = h · I` mod 2,
which is `I` for h odd and **0** for h even — chain-map injectivity
fails in characteristic 2 when `2 | h`. This is the **§6j obstruction
reappearing in homological form** ("characteristic of F₂ divides h" plays
the role of "2 | |G|").

Per the T2R5.0 spec, the question is whether the *empirical conjecture*
(SRB §7) `d ≤ d_h ≤ h·d` is worth implementing as a candidate bound
without the formal h-odd guarantee:

- For gross (h=2 cover of [[72,12,6]] with d=6): the conjecture gives
  `d_gross ≥ 6`. Non-trivial (beats prior rounds' "1" on gross) but
  **not tight** (gap of 6 short of d=12).
- For [[72,12,6]] itself (h=1, trivial self-cover): the conjecture
  is vacuous (d ≥ d), no real bound.
- For BB codes that are *cyclic* / *generalised-toric* covers with **h
  odd**: the lower bound `d_h ≥ d` is rigorous from base distances.

**Decision: implement the SRB conjecture as the candidate bound.** It is
the *first* homological family bound the program has tested that is
both (a) self-contained for individual BB instances (given a known base
code), and (b) gives a non-trivial number on gross. The result will not
be tight on gross (which is the d=12 target); this becomes a
"survives-but-not-tight-on-gross" candidate. The implementation will
record both the rigorous (h-odd) sub-condition and the conjectural
(all-h) form, separately, so the corpus sweep can falsify the
conjecture cleanly.

## 1. The candidate paper

**B. C. B. Symons, A. Rajput, D. E. Browne (2025).**
*Sequences of Bivariate Bicycle Codes from Covering Graphs.*
[arXiv:2511.13560](https://arxiv.org/abs/2511.13560), Nov 17, 2025.
Affiliations: Edinburgh, Oxford, UCL.

### What the paper does

SRB shows that given a base BB code `Q(A, B, l, m)` with parameters
`[[n, k, d]]`, an h-fold covering graph of the Tanner graph corresponds
to another BB code `Q̃(Ã, B̃, l̃ = ul, m̃ = tm)` with `h = ut` and
parameters `[[hn, k_h, d_h]]`, where the cover code's defining
polynomials are subject to simple algebraic conditions
(Theorem 3.1):

- `l̃ = ul`, `m̃ = tm`, and
- `Mod(α̃_{i,1}, l) = α_{i,1}`, `Mod(α̃_{i,2}, m) = α_{i,2}` for each
  monomial of `Ã` and `B̃`.

Practically: each cover monomial reduces (mod `l, m`) to the
corresponding base monomial. Many "interesting" BB codes from the
Bravyi 2024 table (including gross) arise as covers of much smaller
base codes (e.g. gross is a double cover of [[72,12,6]]).

The graph covering map `π : T̃ → T` induces a **chain map** between
the two BB codes' CSS chain complexes
(`Q̃• = (F₂^{l̃m̃} → F₂^{2l̃m̃} → F₂^{l̃m̃})`,
similar for `Q•`), in both directions:

- **Projection** `p• : Q̃• → Q•` (Theorem 4.1) — projects logical
  operators from cover to base.
- **Lifting** `τ• : Q• → Q̃•` (Theorem 4.3) — lifts logical operators
  from base to cover. This is the **transpose** of the projection.

### The main distance theorem (Theorem 4.6, 4.7)

> **Theorem 4.6.** If h is odd, `d_h ≤ h · d`.
>
> **Theorem 4.7.** If h is odd AND `k_h = k`, then `d ≤ d_h`.

Combined: under hypotheses (h odd, k_h = k), the cover code's distance
satisfies `d ≤ d_h ≤ h · d`.

**Where the proofs need `h` odd.** Lemma 4.4 computes
`(p ∘ τ)(x^a y^b) = h · x^a y^b mod (l, m)`. Over F₂, this is `I` if h
is odd, **0 if h is even**. So:

- For `h` odd: `p ∘ τ = I`, so `H_1(p) ∘ H_1(τ) = I` on homology, so
  `H_1(τ)` is injective and `H_1(p)` is surjective. The non-zero
  homology class of a base logical lifts to a non-zero homology
  class in the cover — proves `d_h ≤ h · d` (the lift has weight
  `h · d`). The injectivity on homology also proves `k_h ≥ k`.
- For `h` even: `p ∘ τ = 0` on F₂. Chain-map machinery still defines
  the maps but they no longer combine to an identity, so injectivity
  on homology is not guaranteed. The argument breaks structurally.

The authors explicitly call this out (§7 Conclusion):
> "Our proof techniques for establishing bounds on `k` and `d` break
> down when `h` is even and `h` is the characteristic of F₂."

### The empirical conjecture (paper §7, page 39)

> "We have empirically observed the same behaviour for codes with even
> `h` and conjecture that the parameters of a h-cover BB code satisfy
> `[[n_h = h·n, k_h ≥ k, d ≤ d_h ≤ h·d]]` for any h."

All numerical data in Tables 1–10 of the paper is consistent with this.
No counterexamples are reported.

### How it applies to gross

Per **Example 5** (page 18):

> "The gross code [[144,12,12]] with `(Ã = x³+y+y², B̃ = y³+x+x²,
> l̃ = 12, m̃ = 6)` is a double cover of the [[72,12,6]] code with
> `(A = x³+y+y², B = y³+x+x², l = 6, m = 6)`."

So `h = ut = 2` (with u=2, t=1, since l̃/l = 12/6 = 2 and m̃/m = 6/6 = 1).
The base [[72,12,6]] has `d = 6`. Under SRB:

- Theorem 4.6 / 4.7 do **not** apply (h=2 even).
- Conjecture gives `6 ≤ d_gross ≤ 12`. Matches actual `d_gross = 12`.

The conjecture is **consistent** with gross's known distance but does
not pin it down to 12 by itself. It says `d_gross ≥ 6`, which is
substantially better than prior-round bounds on gross (HT/Roos/BBCS:
trivial 1; Lin-Pryadko Statement 12: 3), but still 6 short of the
target.

### Does the bound require F₂[G] semisimple?

**No, not directly.** The chain-map / cover-graph framework operates
on the bare F₂-vector-space chain complex of the BB code, with no
reference to characters of G or Wedderburn decomposition of F₂[G].
The §6j obstruction (semisimplicity, `gcd(|G|, 2) = 1`) does not
re-apply.

**But it reappears in disguise.** The h-odd hypothesis comes from
`char(F₂) = 2` dividing `h = 2`. For gross-as-double-cover, this
is the same arithmetic obstacle that produced "2 | |G|" in §6j.
The chain-map argument needs `h ≠ 0 mod 2`, equivalently `h odd`,
equivalently "the cover's `F₂`-coefficient does not annihilate the
chain map under composition". So **the gross instance lives precisely
where the standard chain-map argument fails.**

This is the structural-honesty observation of T2R5.0: the homological
family escapes §6j *in general* but does **not** escape it *for gross
specifically* — the gross code occupies the *exact* part of the
construction where the technique breaks. Any homological argument
that bounds `d_gross` from below tight to 12 must either:

1. Find a different base code such that gross is an *odd*-cover of it
   (none reported in the literature; SRB's identification of the
   [[72,12,6]] base is the natural one).
2. Strengthen the chain-map argument to work in characteristic 2 (no
   technique known to the authors).
3. Use a different homological construction entirely (mapping cone,
   spectral-sequence-based, balanced product distance lemmas, etc.) —
   these are conjectured by SRB to give answers but not formalized
   for BB.

## 2. Other candidates from the spec, briefly

### 2a. Wang–Pryadko 2022 (the "real" reference for BB-style distance bounds)

**R. Wang, L. P. Pryadko (2022).** *Distance Bounds for Generalized
Bicycle Codes.* [arXiv:2203.17216](https://arxiv.org/abs/2203.17216),
Symmetry 14(7), 1348.

This is the only published systematic treatment of distance bounds
for GB codes (which subsume BB codes via the abelian-G ring
`F_2[Z_ℓ × Z_m]`). Statements:

- **Statement 1**: distance of `GB(a, b)` equals distance of `AB(a₁, b₁)`
  where `a₁ = a/gcd(a, b)`, `b₁ = b/gcd(a, b)`. Reduction to coprime
  case.
- **Statement 2** (upper bound, Z-distance):
  `d'_Z ≤ d_g`, where `g(x) = (x^ℓ - 1)/gcd(a, b, x^ℓ - 1)` and `d_g`
  is the cyclic-code minimum distance of the code generated by `g`.
  **Useful for *upper*-bounding `d`, not lower-bounding it.**
- **Statement 4** (commensurate case): `[[2ℓ, k, d]] = `direct sum of
  `[[2ℓ₀, k₀, d₀]]` covers a degeneracy classifier we have already.
- **Statement 6** (specialized weight-4 / nice-factoring case):
  pointwise lower bound `d_0 ≥ min(d[q], 1 + d[p])` *under conditions*
  (a) `r(x) = 0` or (b) `gcd(r(x), x^ℓ - 1) = 1`, where the polynomial
  `a(x) = f(x) h(x)`, `b(x) = h(x)`. Restrictive — requires `b` to
  divide `a` and `g(x) = p(x) q(x)` with conditions on `f - r = p`.
  **Does not apply to gross's polynomials** (`b = y³+x+x²` does not
  divide `a = x³+y+y²`; both are weight-3 over `Z_12 × Z_6` without
  any such factorization).
- **Statement 7** (Gilbert–Varshamov-style lower **existence** bound):
  `d_GV` defined by an inequality; **existence** of `f(x)` such that
  `d ≥ min(d[g], d_GV)`. Requires `g(x)` irreducible. **Existence-only,
  not pointwise on a given (a, b)**. Not applicable to gross
  (we already have gross's specific polynomials; existence bounds
  don't help).
- **Statement 13** (upper bound): a GB code with row weight `w` is
  equivalent to a CSS code local in `D ≤ w - 1` dimensions; by
  Bravyi–Poulin–Terhal `d ≤ O(n^{1 - 1/D})` and
  `k · d^{2/(D-1)} ≤ O(n)`. For gross with w=6: `D ≤ 5`,
  `d_gross ≤ O(144^{4/5}) ≈ 53`. Consistent with d=12 but very loose;
  **upper, not lower**.
- **Statement 14** (only pointwise lower bound in the paper, weight-4
  only): a weight-4 GB code of odd distance `d = 2r + 1` has length
  `n ≥ 1 + d²`; for even distance `n ≥ d²`. **Restricted to weight-4
  GB codes**. Gross is weight-6. Not applicable.

The only candidate from Wang–Pryadko that gives a per-instance lower
bound on gross is **Statement 6**, which has restrictive structural
preconditions that gross fails.

**Does W-P require `F_2[G]` semisimple?** Statement 7 explicitly needs
`g(x)` irreducible, which requires `ℓ` odd (so that `x^ℓ - 1`
factors squarely). For gross (ℓ=12, m=6, both even), this fails.
Statements 1, 2 only require `ℓ` finite — no semisimplicity.
Statement 13 is dimension-based, no semisimplicity. The relevant
statements for *gross-shape* polynomials therefore split:
"semisimplicity needed" (Stmt 7 — GV existence) vs. "semisimplicity
NOT needed" (Stmts 1, 2, 13) — but only Stmt 6 is a per-instance lower
bound, and it has its own restrictive structural condition.

### 2b. Kovalev–Pryadko 2013 ([arXiv:1212.6703](https://arxiv.org/abs/1212.6703))

Original "hyperbicycle" construction. The paper's distance bounds
(in their §III–IV) are **square-root-of-n existence** bounds for
families. No per-instance lower bound is given.

The paper is the *constructive* origin of the hyperbicycle/GB
families; W-P 2022 above is the dedicated distance-bound follow-up.
The latter subsumes the former for our purposes.

### 2c. Hsieh–Le Gall 2020

The arXiv ID 2009.09690 (in the spec) does **not** match a Hsieh–Le Gall
paper on QECC — it's a paper on elasticity / rank-one convexity in
PDEs. The correct reference is likely M.-H. Hsieh's work on quantum
codes; nothing turns up specifically on BB codes. The recent (2025)
Hsieh "Simplified Quantum Weight Reduction" paper
([arXiv:2510.09601](https://arxiv.org/abs/2510.09601)) is about
weight-reduction transforms, not lower bounds on BB distance.

### 2d. Tillich–Zémor 2014, Hastings 2017, Panteleev–Kalachev 2021

These are the **upper-distance / construction** families (hypergraph
product, fiber-bundle, lifted-product). They prove `d ≥ √n`
existence-style for new constructions; they do **not** give per-instance
distance lower bounds for an arbitrary specified BB pair `(A, B)`.

## 3. Why SRB 2025 (with the empirical conjecture, declared honestly) is
the right candidate for T2R5

Three reasons:

1. **Homological by construction.** The proof is entirely on chain
   complexes / cover-graph maps; no character-variety, Wedderburn,
   or Jacobson appears anywhere. The §6j obstruction is not the
   blocker — instead the blocker is a *new* one, "h coprime to
   char(F₂) = 2", which we should document as §6k.

2. **BB-specific, not just general LDPC.** SRB explicitly works with
   the bivariate-bicycle CSS chain complex `Q• : F₂^{lm} → F₂^{2lm}
   → F₂^{lm}` parameterized by `(A, B, l, m)`, identical to
   `BBChainComplex.lean` in our Lean side. Theorem 3.1 + Example 5
   identify gross as an explicit double cover of [[72,12,6]].

3. **Computable per-instance.** Given `(A, B, G)` of a candidate cover
   code, the algorithm:
   - Search divisors `(u, t)` of `(l, m)` with `h = ut > 1`.
   - For each, project (mod `(l/u, m/t)`) the polynomials to candidate
     base codes.
   - Check whether the projected `(A', B')` define a valid base BB code
     (CSS commutation, well-defined).
   - For each valid base, look up or compute its distance `d_base`.
   - Apply: `d ≥ d_base` (rigorous if h odd & `k_h = k`, conjectural
     otherwise) and `d ≤ h · d_base` (same conditions).

   The bound's *value* is `max over valid bases of d_base` (best lower
   bound). The condition `S(A, B)` is "exists a valid odd-h base
   cover whose distance we know".

The deliverable for T2R5.1 is therefore:

- `bb_homological_bound(A, B, G, base_distance_lookup)` → int.
- `bb_homological_condition(A, B, G)` → bool (whether a valid
  odd-h base exists).

Computability: small abelian groups (gross's `Z_12 × Z_6`) have
O(d(l) · d(m)) divisor pairs, each with O(|G|) factoring work — easily
sub-second on each instance.

## 4. Property checklist for SRB 2025

| Property | SRB 2025 | Evidence |
|---|:---:|---|
| (a) Non-cyclic G (gcd(ℓ, m) > 1) | **YES** | Throughout — paper works directly on `Z_ℓ × Z_m`. |
| (b) Non-semisimple F₂[G] (2 \| \|G\|) | **YES** | No semisimplicity used. All Theorems 3.1, 4.1, 4.3 work on any abelian group. |
| (c) Per-instance pointwise lower bound | **YES (if h odd)** | Theorem 4.7. Requires base code with known distance. |
| (d) Per-instance pointwise lower bound on gross | **NO** | h=2 even, falls into the conjectural regime. |
| (e) Empirical conjecture covers gross | **YES** | Paper §7 conjecture; consistent with d=12. |
| (f) Tightness on gross | **NO** | Conjecture gives 6 ≤ d ≤ 12. Loose by 6 from below. |

Property (b) is the win over §6j. Property (d) reveals the new
obstruction (call it §6k below) — there is no proven lower bound on
gross from SRB, only a conjecture.

## 5. The new §6k obstruction (proposed)

For HANDOFF.md if this round shelves:

> **§6k. Cover-graph chain-map bounds require h coprime to char(F₂);
> gross is a 2-cover, where the chain-map technique fails.**
>
> SRB 2025 (arXiv:2511.13560) gives a per-instance lower bound
> `d_h ≥ d_base` whenever the BB code is an h-fold Tanner-graph cover
> of a smaller BB code, **provided h is odd**. The hypothesis is
> structural: the proof requires `p ∘ τ = h · I` to equal `I` mod 2
> (Lemma 4.4), which holds iff h is odd. Equivalently, the chain map
> τ_• is injective on F₂-homology iff `h ∉ char(F₂) · Z`.
>
> Gross [[144,12,12]] is a **double cover (h=2)** of [[72,12,6]]
> (SRB Example 5). The hypothesis fails. The paper conjectures the
> bound still holds for even h based on extensive numerical data, but
> no proof exists for the h-even case.
>
> Implication: under SRB's rigorous theorem, the lower bound on
> `d_gross` is the *trivial* 1 (no usable base code with h odd). Under
> the empirical conjecture, `d_gross ≥ 6` (from base [[72,12,6]]).
> Neither gets to the target d=12.
>
> This is the homological analogue of §6j's "2 | |G|" obstruction:
> the *same arithmetic 2-divisibility* that prevented Fourier-transform
> bounds in §6j now blocks the chain-map injectivity argument. The
> obstruction is family-specific, not just paper-specific: any
> chain-map technique that requires "lifting map cross projection map
> equals identity" will be 2-blind on the gross instance.
>
> **Rule for future Tier-2 candidates**: when a homological technique
> requires "h coprime to char(F)" or "lifting map injective mod p",
> verify whether gross falls into the working regime before
> committing. If gross has h = 2 (or more generally, if the relevant
> homological-degree multiplier is even), the bound's value on gross
> is determined by the surviving auxiliary structure, not by the
> primary theorem.

## 6. Verdict

**Per the T2R5.0 spec, the literature check resolves to:**
**proceed with the SRB 2025 candidate**.

The candidate satisfies the spec's bar: (a) it is homological, (b) it
gives a value > 1 on gross (under the conjecture), (c) the derivation
does not pass through any character-theoretic step that would
re-instate §6j.

But the candidate is **a-priori known to be loose on gross**: the
rigorous theorem doesn't apply (h=2 even), and the conjectural form
gives only `d_gross ≥ 6`, half the target.

Given the §6h-§6j discipline, the honest expected outcome is:

- **survives-but-trivial-on-gross** at best (`6 ≤ bb_homological_bound
  ≤ 8`).
- **shelved-paper-doesnt-help** if the corpus sweep reveals violations
  (which would falsify the SRB *conjecture*, an interesting outcome).

Either way, the corpus sweep + Bravyi-table check is the right next
step. Implementing the bound is approximately a half-day; running it
across the corpus is minutes.

The §6k obstruction articulation is the *useful negative output* of
this round even if the bound shelves, since it characterizes the
new family of obstructions that any future homological candidate
must dodge.
