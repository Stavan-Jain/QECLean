# T2R2.1 — Inventory of weight-vs-dimension invariants

Date: 2026-05-26. This is the first inventory pass of Tier-2 Round 2,
written to discipline future bound proposals against the "no dimension
counts on the RHS of `d ≥ …`" rule that killed Tier-2 Round 1's
Jacobson-radical conjecture (see HANDOFF.md §6h).

## The rule (recap)

A **weight invariant** is a quantity whose value changes if the
underlying linear code's weight enumerator changes, holding the code's
dimension constant. Concretely, if you replace `(A, B)` with another
pair `(A', B')` whose `M_A, M_B` have the same dimensions for
`ker`, `image`, `cokernel`, etc., but the minimum-weight non-zero
elements of those subspaces differ, a weight invariant *changes value*.
A **dimension invariant** stays constant under such substitution.

Weight invariants can bound `d` from below: the minimum-weight argument
for any nonzero logical translates directly. Dimension invariants
bound `k` (the dimension of the logical-coset space), not `d`.

## Weight invariants ALREADY computable on a row (no new code)

1. `min(min_wt_ker_A, min_wt_ker_B)` — the canonical `min(d_A^⊥, d_B^⊥)`.
   The dominant Tier-2 predictor per T2.1 classifier (94% Gini).
2. `tanner_girth` — combinatorial girth of the bipartite Tanner graph.
3. `A_weight + B_weight` — sum of polynomial Hamming weights = row weight
   of `H_X` (note: this is an *upper* bound on `d`, not a lower bound).
4. **Per-orbit min-weight of `ker M_A`**: not yet computable — would
   require enumerating the kernel restricted to each isotypical component
   of `F_2[G]`. The aggregate `min_O d_O^⊥(A) = d_A^⊥ = min_wt_ker_A`
   already in the corpus is the *minimum* over orbits, but per-orbit
   values are richer (one of them may equal `d_A^⊥`; the others are
   typically larger).

## Weight invariants that need NEW code

This is the work for T2R2.2. Each item below MUST be a weight invariant
(passes the rule's "isomorphic-codes-different-weight-enumerators" test).

1. **`per_orbit_dual_distance(poly, G)`** — for each Frobenius orbit O
   on Ĝ where `poly` vanishes (i.e. its image in the corresponding
   factor of the semisimple quotient is zero), compute the minimum
   Hamming weight of any nonzero element of `ker(M_poly)` that's
   supported on the isotypical component for O.

   This is a *weight invariant*: changing the polynomial values in F̄₂
   while preserving the dimension of each isotypical kernel summand
   would change the per-orbit minimum weight.

   The aggregate `min_O d_O^⊥(poly)` equals `d_A^⊥` = `min_wt_ker_A`
   already in the corpus, but the per-orbit breakdown is new and
   strictly richer.

2. **`tz_lower_recomputed(A, B, G)`** — Lin-Pryadko Statement 12
   `⌈min(d_A^⊥, d_B^⊥) / c⌉` packaged as a feature (not just a
   candidate). `c = |G_a ∩ G_b|` where `G_a = ⟨supp(A)⟩` and similarly
   for B. Pure weight invariant on the numerator (the dual distances)
   divided by a structural count (the intersection-subgroup order).

3. **`bch_per_orbit_lower_bound(poly, G)`** — BCH-style designed distance
   applied to each cyclic component of the abelian-G group algebra. For
   `G = Z_ℓ × Z_m` and a polynomial vanishing on a chain of consecutive
   characters within an orbit, the BCH bound gives `d_O^⊥ ≥ δ + 1`
   where `δ` is the number of consecutive vanishing characters. The
   weakest BCH bound over orbits is a lower bound on `min_wt_ker_A`.

4. **`joint_support_min_distance(A, B, G)`** — for the joint-vanishing
   set, the minimum Hamming weight over `ker M_A ∩ ker M_B`. NOT THE
   DIMENSION — the minimum-weight element. This is a weight invariant
   of the joint kernel as a linear code.

   *Caveat*: this overlaps with `d_X` itself (the joint kernel contains
   logical-X coset reps), so it's an upper bound on `d_X`, not a lower
   bound. Useful diagnostically; not RHS material for `d ≥ …`.

## Per-orbit dual distance: definition (preview for T2R2.2)

For polynomial `A ∈ F_2[G]` and Frobenius orbit `O ⊂ Ĝ`, define the
**isotypical component** of `ker(M_A)` for `O`:

```
ker_O(M_A) := { x ∈ ker(M_A) : χ̂(x) = 0 for all χ ∉ O̅ }
```

where `O̅` is the Frobenius-closed subset of characters not in O. (In
the semisimple case, this is just the projection onto the F_{2^|O|}
component of F_2[G].)

The **per-orbit dual distance** is

```
d_O^⊥(A) := min { |x| : x ∈ ker_O(M_A) \ {0} }
```

Properties:
- `d_O^⊥(A) ≥ 1` always (a nonzero kernel element has at least one
  nonzero coordinate).
- For orbits where A *does not* vanish, `ker_O(M_A) = {0}` and
  `d_O^⊥(A) = +∞` (no nonzero vectors).
- `min_O d_O^⊥(A) = d_A^⊥ = min_wt_ker_A` (already in corpus).

This is the natural per-orbit refinement of the classical-dual minimum
distance.

## What is NOT a weight invariant (don't put on RHS)

- `dim_ker_A`, `dim_ker_B`: rank-deficits = dimension counts.
- `μ_O(A)`, `Σ |O| · μ_O`: Jacobson-radical depth and sums.
  **Already falsified** as RHS in T2R1.
- `n_vanishing_orbits(A)`, `|joint vanishing|`: structural counts.
- `k`, `n - rank_HX`: ditto.
- `orbit_size`: pure group structure.
- `supp_diameter_*`: structural feature, no weight content.

Using any of these as the RHS of a `d ≥ …` bound directly repeats
Round 1's mistake.

## What CAN be on the RHS (the "OK list")

In increasing order of theoretical strength:

1. **`min_wt_ker_A`, `min_wt_ker_B`**: well-defined dual-classical
   minimum distances. The textbook **upper** bound `d ≤ min(d_A^⊥, d_B^⊥)`
   uses these — but this is an upper bound, not a lower bound (HANDOFF §6a).
2. **`tanner_girth`**: combinatorial weight invariant; ldpc theory gives
   weak lower bounds via stopping sets.
3. **`⌈d_A^⊥ / c⌉, ⌈d_B^⊥ / c⌉`**: Lin-Pryadko Statement 12 lower bound.
4. **`d_O^⊥(A)` per orbit**: new candidate space — per-orbit refinements
   of (3) and (1).
5. **Joint-orbit dual distances**: weight invariants of the joint kernel.

The Round-2 candidate space is built from (3), (4), (5).

## Decision: T2R2.2 will implement

- `per_orbit_dual_distance(poly, G)` — returns `dict[orbit -> int]`,
  the per-orbit dual distance (or `+∞` for non-vanishing orbits).
- `tz_lower_recomputed(A, B, G)` — a clean callable for
  `⌈min(d_A^⊥, d_B^⊥) / c⌉` (refactor of the lambda in
  `scripts/tier2_candidates_lit.py`).
- `bch_per_orbit_lower_bound(poly, G)` — BCH/HT designed-distance applied
  per orbit, returning a lower bound on `d_A^⊥` derived from algebra
  (not from kernel enumeration).
- `joint_kernel_min_weight(A, B, G)` — diagnostic, an upper bound on
  d_X via the joint kernel.

All four are weight invariants. None are dimension counts.

## Caveat: hybrid features

A few features are *hybrid* (mix weight and dimension) and need care:

- `min(d_A^⊥, d_B^⊥) / c`: numerator is a weight invariant, denominator
  is a structural (subgroup-order) count. The ratio is a weight invariant
  *per orbit* if we use a per-orbit c, but with the global c it has a
  structural denominator. The literature uses it; we follow.
- `min_O d_O^⊥(A) / r(A)` where `r(A)` is the "orbit cover radius"
  of the support: weight numerator, structural denominator. Same shape.
- Per-orbit min-weight times a dimension count would be a tradeoff
  shape — useful in principle but to be avoided as the "main" RHS.

The rule of thumb: the **numerator** must be a weight invariant. The
**denominator** can be structural (a `min` or `floor` over a discrete
ratio is still a weight quantity, just a coarser one).
