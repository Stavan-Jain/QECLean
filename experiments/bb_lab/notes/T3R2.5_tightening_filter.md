# T3R2.5 — Tightening filter exploration

The 43 non-degenerate violators of the real Jacobson bound (T3R2.2)
share a uniform structural property (T3R2.3 §5): each admits a
**weight-≤2 algebraic syzygy** in `F_2[G]²`, i.e. an `(α, β)` of
total Hamming weight ≤ 2 with `αA + βB = 0`. This note explores
whether forbidding that syzygy gives a clean conditional conjecture.

---

## 1. The candidate filter

> **TIGHTENING** :≡  no weight-≤2 element `(α, β) ∈ F_2[G]²` with
> `αA + βB = 0`.

Equivalently, three sub-conditions, any one of which signals the
filter is "rejected":

* `min_wt_ker_A ≤ 2`  (A has a weight-2 dual codeword on the A-block).
* `min_wt_ker_B ≤ 2`  (B has a weight-2 dual codeword on the B-block).
* `B = unit · A` in `F_2[G]`  (a "mixed" weight-2 syzygy of the form
  `(e_g, e_h)` exists).

A code passes the filter iff none of the three triggers.

## 2. Performance on the strict-non-degenerate corpus

Applied to all 1937 non-degenerate corpus rows (strict definition):

| | rows | violations of real Jacobson |
|---|---:|---:|
| non-degenerate only | 1937 | 43 |
| non-degenerate AND tightening | **1796** | **0** |

The filter rejects exactly the 141 non-degenerate rows with
`d_exact = 2`. Among those, 43 were violators and 98 were tight
non-violators (where `bound = 2 = d_exact`). All `d_exact ≥ 4`
non-degenerate codes pass the filter.

Filter rejection breakdown over 1937 non-degenerate rows:

| reason | count |
|---|---:|
| mixed-monomial (`B = u · A`) | 139 |
| block-B-dual-2 (`min_wt_ker_B ≤ 2`) | 2 |
| block-A-dual-2 (`min_wt_ker_A ≤ 2`) | 0 |

The 139 / 2 split closely matches the violator split (41 / 2), with
the gap being non-violator d=2 codes that incidentally have unit-translate
B but are still tight.

## 3. Performance on the Bravyi table

The bigger problem: **all 5 Bravyi instances are degenerate** under the
strict non-degenerate definition (T3R2.4 §1). The filter is *applied
within* the non-degenerate condition, so it makes no statement about
Bravyi.

If we instead use the looser **joint** non-degeneracy condition
`⟨supp(A) ∪ supp(B)⟩ = G` (under which all 5 Bravyi codes are
non-degenerate), the picture is:

| code_id | joint nondeg | passes tightening | bound | d | slack |
|---|:---:|:---:|---:|---:|---:|
| bb_72_12_6  | yes | **yes** | 8 | 6 | -2 |
| bb_90_8_10  | yes | yes | 4 | 10 | +6 |
| bb_108_8_10 | yes | yes | 4 | 10 | +6 |
| gross       | yes | yes | 8 | 12 | +4 |
| bb_288_12_18| yes | yes | 16 | 18 | +2 |

Under the joint-degenerate-plus-tightening combination,
**bb_72_12_6 violates and passes the tightening filter** — so the
tightening doesn't catch bb_72_12_6 either. It's a different failure
mode.

### Why bb_72_12_6 is not caught

bb_72_12_6 has:
* `min_wt_ker_A = 6` (≥ 3, passes)
* `min_wt_ker_B = 6` (≥ 3, passes)
* `B ≠ unit · A` (passes the mixed-monomial check)

So the polynomial-side weight-2-syzygy filter doesn't reject it. But
the SAT-true distance is 6, the conjecture's RHS is 8, slack = -2.
The over-count comes from the dimension-of-Σ|O|·μ_O sum:

| orbit | size | μ_A | μ_B | contrib |
|---|---:|---:|---:|---:|
| `{(1,1), (2,2)}` (dim-3 cube-root char) | 2 | 2 | 2 | 4 |
| `{(1,2), (2,1)}` (other cube-root) | 2 | 2 | 2 | 4 |
| **total** | | | | **8** |

But `dim(ker M_A ∩ ker M_B) = 6` (computed directly), which equals
`k/2 = 6` (Bravyi Lemma 1) and equals `d_exact`. The conjecture's
RHS **overshoots the joint-kernel dimension itself** in this
non-semisimple case — the orbit-wise sum `Σ|O|·min(μ_A,μ_B)` is
larger than `dim(ker M_A ∩ ker M_B)`. This is a structural defect
of the conjecture even at the dimension-count level, not just a
weight-vs-dimension mismatch.

## 4. Implications

The tightening filter is **clean on the strict-non-degenerate
corpus**: zero violations on 1796 rows. But it doesn't extend to
the Bravyi target via the joint-non-degeneracy alternative, because
bb_72_12_6 violates via a *different* structural mechanism (deep
radical over-count, not low-weight syzygy).

Possible refinements (none tested in this session):
* Add a fourth sub-condition: `Σ|O|·min(μ_A,μ_B) ≤ dim(ker M_A ∩ ker M_B)`
  to ensure the bound never exceeds the actual joint-kernel dimension.
  This would catch bb_72_12_6. But it's a self-referencing post-hoc
  patch ("the bound holds whenever the bound is no larger than the
  thing it tries to bound"), not a clean conditional theorem.
* Move to per-orbit local bounds instead of a single sum: investigate
  whether `d ≥ min_O (|O| · min(μ_A,μ_B))` (a min, not a sum) survives
  bb_72_12_6 — initial check suggests yes (the orbit-wise contribution
  is 4 for both orbits, ≤ 6 = d). But this would be a *different*
  conjecture, not a tightening of the original.

For this session: report the filter, note it does NOT cover Bravyi,
and recommend no Tier-4 work without further structural insight.

## 5. Summary headline numbers

| filter | rows in scope | violations | clean? |
|---|---:|---:|:---:|
| nondegenerate (strict) | 1937 | 43 | no |
| nondegenerate (strict) ∧ no weight-2 syzygy | 1796 | 0 | yes (on corpus) |
| nondegenerate (strict) ∧ no weight-2 syzygy ∧ Bravyi | 0 | n/a | trivially yes (empty domain) |
| nondegenerate (joint) | n/a (not measured on corpus) | bb_72_12_6 violates | no |
| nondegenerate (joint) ∧ no weight-2 syzygy | n/a | bb_72_12_6 still violates | no |

The combination "strict non-deg + no weight-2 syzygy" is empirically
clean on the corpus, but **its domain excludes the engineering target
(Bravyi family)**. There is no candidate filter that's clean on the
corpus AND covers Bravyi.
