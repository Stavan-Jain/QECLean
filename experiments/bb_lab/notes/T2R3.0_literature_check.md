## 1. Univariate Hartmann–Tzeng (HT) and Roos — confirmed prior art

**Hartmann & Tzeng 1972** (van Lint, Tzeng-Hartmann line; the classical
generalization of the BCH bound):

> For a univariate cyclic code `C ⊆ F_q[x]/(x^n - 1)` with defining
> set `T ⊆ Z_n`, if `T` contains the AP-pattern
> `{a + i s + j s' : 0 ≤ i ≤ δ−2, 0 ≤ j ≤ r}`
> with `gcd(s, n) = 1` and `gcd(s', n) ≤ δ − 1`, then `d(C) ≥ δ + r`.

For `r = 0` this reduces to the classical BCH bound.

**Roos 1983**: further generalization with a non-arithmetic shifted
pattern. Improves HT when the defining set is more sparse than an AP.

Implementations and refinements:
- Camion 1970 — "apparent distance" of abelian codes, generalizing
  BCH to multivariate-cyclic.
- Sabin & Lomonaco 1992 — multivariate cyclic codes.
- Saints & Heegard 1995 (TIT) — "Algebraic-geometric codes and
  multidimensional cyclic codes: A unified theory and algorithms for
  decoding using Gröbner bases".
- Bernal, Bueno-Carreño & Simón 2016 (TIT 62, 655–668) "Apparent
  Distance and a Notion of BCH Multivariate Codes" (arXiv:2402.03938
  is the 2024 preprint update / extension). Reduces bivariate Camion
  computation from exponential to linear via hypermatrix manipulation.
- Other extensions: Roos bound improvements (van Lint–Wilson, Betti–Sala).

All five are real classical-coding-theory references. The Bernal et al.
algorithm is the practical computational tool: it computes the apparent
distance of an abelian code in `F_2[Z_n1 × Z_n2 × ...]` from its
**defining set** (the set of vanishing characters), and the apparent
distance is a lower bound on the classical minimum distance.

## 2. HT/Roos applied to BB codes — NOT in the literature (to my knowledge)

I searched (May 2026) for:
- "bivariate bicycle code Hartmann-Tzeng bound" — no hits
- "BB code Roos bound" — no hits
- "two-block group algebra Camion apparent distance" — no hits
- "Coprime Bivariate Bicycle Codes Hartmann-Tzeng" — no hits

The five primary BB-code papers (LP23, Bravyi 2024, KP13,
Panteleev-Kalachev 2021, Wang-Pryadko 2022) and the recent papers
that came in this search (Wang–Mueller 2024 *Coprime BB Codes*
arXiv:2408.10001; Postema–Kokkelmans 2025 *Existence and
Characterisation of BB codes* arXiv:2502.17052; *Sequences of BB Codes
from Covering Graphs* arXiv:2511.13560) **do not cite or use** HT,
Roos, Camion, apparent distance, cyclotomic-coset arguments, or
multivariate-cyclic-code defining-set bounds.

The closest is **LP23 §IV.F Statement 12** (Tillich–Zémor analog
lower bound), which is `d ≥ ⌈min(d_A^⊥, d_B^⊥) / c⌉` with
`c = |G_a ∩ G_b|`. This uses **classical** (per-block) dual distances
`d_A^⊥`, not multivariate-cyclic apparent distance of `A` as a
2D cyclic code in `F_2[G]`. The classical-dual-distance interpretation
`d_A^⊥ = min wt(ker M_A)` is a property of the matrix `M_A` (the
circulant convolution operator), not of `A` viewed as an element of
the cyclotomic group algebra `F_2[G]`. So HT/Roos-style bounds on
`A` as a 2D cyclic code are a **strictly different (and potentially
sharper)** object than `d_A^⊥`.

### Why is HT/Roos for BB codes potentially interesting?

The polynomial `A ∈ F_2[Z_ℓ × Z_m]` defines a **2D cyclic code** `C_A`
(the principal ideal `⟨A⟩` in `F_2[G]`) and its dual `C_A^⊥`.
HT/Roos / Camion's apparent distance provides a **lower bound** on
`d(C_A^⊥)`. The textbook CSS upper bound (HANDOFF §6h) says

> `d_X(BB(A,B)) ≤ min(d^⊥(C_A), d^⊥(C_B)) = min(d_A^⊥, d_B^⊥)`.

So combining: `mv_ht(T_A) ≤ d_A^⊥`. This is **NOT** automatically
a lower bound on `d_X` — see Task T2R3.3 below for the gap.

### Does HT actually give a *positive* bound on `d_X`?

Not directly. HT gives `d^⊥(C_A) ≥ mv_ht(T_A)`. The textbook bound
above gives `d_X ≤ min(d_A^⊥, d_B^⊥)` (an upper bound). Composing
these two gives nothing for the lower-bound direction. To get a
positive lower bound on `d_X`, we'd need an additional structural
argument that **the X-logical of minimum weight is dominated by
a single-block witness** in `ker(M_A^T)` or `ker(M_B^T)`. That is
the question we'll attempt to characterize in T2R3.3.

The literature does have a partial single-block argument: LP23
Statement 5 (the "δ_X = δ_Z = 0" case) gives `d ≥ d(A, B^T)` where
`d(A, B^T)` is the *quantum* distance of the half-sized erasure code
— not a closed-form formula. We're after a closed-form replacement
of `d(A, B^T)` via HT/Roos.

## 3. Verdict

| Question | Answer | Evidence |
|---|---|---|
| Is HT/Roos for multivariate cyclic codes a real construction? | YES (since 1972/1983) | Hartmann-Tzeng 1972; Roos 1983; Camion 1970 multivariate generalization |
| Is the bound computable for our `G` shapes? | YES, computable in `poly(|G|)` time | Bernal et al. 2016 algorithm; brute-force is also fine for `|G| ≤ 200` |
| Has it been applied to BB codes? | NOT in surveyed literature | Searched, no hits in LP23 / Bravyi / KP13 / PK / WP / Otjens / Wang-Mueller / 2511.13560 / EC Zoo / Bernal et al. |
| Does HT give a lower bound on `d_X`? | NO, directly. Only on `d^⊥(C_A)` | Composing HT ≤ `d_A^⊥` with `d_X ≤ d_A^⊥` is trivially compatible but yields no lower bound on `d_X` |
| What's needed to get `d_X ≥ mv_ht(...)`? | Structural condition `S(A, B)` that single-block witnesses dominate | TBD in T2R3.3 |

**Novelty assessment**: "applying multivariate HT/Roos / Camion's apparent
distance bound to BB codes, combined with a structural single-block
dominance argument" is **novel-to-us** with high confidence and
**plausibly novel-to-literature** with medium confidence. The
ingredients are textbook (HT 1972, Roos 1983, Camion 1970, CSS upper
bound 1996). The composition has not been published.

If a structural condition `S(A, B)` ensuring single-block dominance
turns out to be: (a) provable as a theorem (closed-form check), and
(b) satisfied by Bravyi's gross polynomials, then `d ≥ mv_ht(T_A) =
12` (or matching/close to) would be a publishable result.

If `S(A, B)` is provable but doesn't fire on gross (e.g., requires
non-degeneracy, repeating the §6i failure mode), this would be a
"survives-loose-on-gross" outcome — useful for understanding BB codes
broadly but not closing the gross gap.
